TP : Gestion d'un Panier E-commerce (POO + MySQL)

Contexte

Vous développez le back-end d'un site de vente en ligne. Chaque produit a des règles de TVA et de frais de port différentes selon son type (livre physique vs ebook).
Vous allez modéliser ça en POO (héritage + polymorphisme), puis lire les produits depuis MySQL pour les transformer en objets et calculer un panier.
Prérequis : POO de base (classes, propriétés, méthodes), notions de PDO. MVC non requis.

Objectifs

Pédagogique : héritage (factoriser le code commun), polymorphisme (traiter tous les produits de la même façon).
Opérationnel : lire des produits en BDD → les instancier → calculer un total de panier sans tester leur type.
Compétence clé visée : comprendre pourquoi le polymorphisme évite les
if
/
switch
à répétition.

Déroulé

Étape 0 — La base de données (fournie)

Créez la base et la table, puis insérez les données.
CREATE DATABASE IF NOT EXISTS boutique CHARACTER SET utf8mb4; USE boutique; CREATE TABLE produit ( id INT AUTO_INCREMENT PRIMARY KEY, nom VARCHAR(255) NOT NULL, prix_ht DECIMAL(8,2) NOT NULL, type ENUM('livre', 'ebook') NOT NULL ); INSERT INTO produit (nom, prix_ht, type) VALUES ('Harry Potter', 20.00, 'livre'), ('Le Seigneur des Anneaux', 25.00, 'livre'), ('Apprendre PHP', 10.00, 'ebook'), ('Maîtriser SQL', 15.00, 'ebook');
​
La colonne

est la clé du polymorphisme : c'est elle qui décidera plus tard quelle classe instancier.

Classe utilitaire en singleton : une seule connexion partagée.
class Database { private static ?PDO $instance = null; public static function getConnexion(): PDO { if (self::$instance === null) { echo "Création de la connexion...\n"; // témoin, à supprimer après test // si t'as pas supprimé ce commentaire, t'as copié collé bêtement, et donc // t'as pas compris ce que tu faisais. self::$instance = new PDO( 'mysql:host=localhost;dbname=boutique;charset=utf8mb4', 'root', '', [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION] ); } return self::$instance; } }

Le

Étape 6 — Le panier polymorphe (pseudo-code)

Le problème : ajouter n'importe quel produit et afficher son prix, sans jamais demander "es-tu un livre ou un ebook ?".
Pseudo-code :
FONCTION ajouterAuPanier(produit : Produit) : afficher "[nom] — TTC : [prixTTC] € — Port : [frais] €"
​
PROGRAMME PRINCIPAL : connexion ← Database::getConnexion() lignes ← exécuter "SELECT * FROM produit" total ← 0 POUR CHAQUE ligne DANS lignes : produit ← creerProduit(ligne) // étape 5 ajouterAuPanier(produit) // polymorphisme total ← total + prixTTC + fraisDePort FIN POUR afficher "TOTAL : [total] €"
​

Points de vigilance / erreurs fréquentes

Oublier
abstract
sur la classe ou la méthode → PHP n'oblige plus les enfants à implémenter.
Mettre les propriétés en
private
au lieu de
protected
→ inaccessibles dans les classes filles.
Écrire un
if ($type === 'livre')
dans le panier → polymorphisme raté, tout l'intérêt s'effondre.
Recréer une connexion PDO dans chaque fonction au lieu d'utiliser le singleton.
